# 音乐编年史报告写作指南（像素级标准）

> 本指南是写作报告时的**强制性规范**。每一条都必须遵守。
> 参考范例：一份3438首歌、11年收藏的完整报告（6800字）。

---

## 一、报告质量底线

### 字数要求（硬性）
- 歌单 < 500首：3000-4000字
- 歌单 500-2000首：4000-6000字
- 歌单 2000-5000首：6000-8000字
- 歌单 > 5000首：8000-10000字

**绝对不允许出现"水字数"的段落。** 每一段必须有独特的洞察、具体的数据支撑或画面感极强的场景描写。

### 深度要求
- 报告不是"你喜欢XX类型的音乐"这种表面描述
- 报告要回答的核心问题是：**这个人为什么在这个时间点选择了这首歌？这个选择揭示了他灵魂的什么秘密？**
- 每一个歌手/歌曲的出现都必须被"解释"——不是介绍这首歌是什么，而是分析它对这个人意味着什么

---

## 二、报告结构（7个章节，严格按序）

### 第1章：音乐内核总述（400-600字）

**目标**：用一个核心比喻统领全文，让读者在前3段就被"击中"。

**必须包含**：
1. 一个具象化的灵魂比喻（不是"你的音乐品味很丰富"，而是"一个人站在一间巨大的空房间中央，穹顶高得看不见"）
2. 总歌数 + 总年份 + 起止年龄的一句话概括
3. 第一首歌的切入——从最早收藏的歌曲揭示"灵魂底色"
4. 11年（或N年）的一句话总结，形成贯穿全文的母题
5. 一个**粗体金句**作为本章结尾（如"**你从来没有在'听音乐'，你一直在'回家'。**"）

**写作技巧**：
- 开篇第一句就要有"锚定感"——一个具体的、可视化的场景
- 然后迅速用3-5个歌手名字证明这个判断
- 最后用一个排比句收束（"XX是庇护所、YY是庇护所、ZZ是庇护所"的结构）

**范例开篇句式**：
```
你的灵魂有一根贯穿始终的弦——**它震颤在"宏大"和"私密"之间，永远不肯只选一边。**
```

```
3438首歌，11年，从21岁到32岁。如果把它们全部碾碎，重新提炼出一种气质，那大概是这样的：
```

---

### 第2章：编年史纪元（每段400-1000字，共3-6个纪元）

**纪元划分原则**：
- 不是机械地每2年一段，而是根据**收藏行为的拐点**划分
- 收藏量骤增/骤降的年份应该成为纪元分界线
- 每个纪元内部应该有统一的主题（如"探索""确认""沉默""回望""和解"）

**纪元标题格式**：
```
## 第X纪元：YYYY-YYYY ——[7字以内的诗意概括]
```

**每个纪元必须包含的6个要素**：

1. **数据开头**（加粗，一句话）：
   ```
   **23-24岁。1194首歌。这是你歌单上最密集的两年，加起来占了总量的35%。你不再是什么都尝的人了——你在确认。**
   ```

2. **关键日期故事化**：
   - 从 `peak_days` 中挑出该纪元内的高频收藏日
   - 用"X年X月X日，你一天收了N首XXX"的方式切入
   - 然后推测当时的场景和心理
   - 范例："2017年2月25日，你一天收了32首动漫歌：灌篮高手、数码宝贝、犬夜叉。这不是怀旧——**这是告别。**"

3. **核心歌手深度解读**（每个纪元2-4个歌手）：
   - 不是"你喜欢XX"，而是"XX给了你什么许可"
   - 必须具体到某首歌的歌名、时长、特征
   - 必须解释"为什么是这个年纪"才需要这个歌手
   - 范例："X JAPAN给了你一种许可：**情绪可以巨大，爱可以巨大，痛也可以巨大。**"

4. **音乐功能转变**：
   - 每个纪元的音乐在用户生活中扮演的角色是什么？
   - 是"探索地图"还是"身份确认"还是"创伤修复"还是"日常陪伴"？
   - 范例："**音乐从地图变成了药箱。** 25岁之前你用音乐画版图，25岁之后你用音乐疗伤。"

5. **与上一纪元的关系**：
   - 递进？转折？延续？断裂？
   - 用一句话交代清楚（如"从'什么都想听'变成'我知道我需要什么'"）

6. **纪元结尾金句**（加粗）：
   - 每个纪元必须以一个总结性的金句结尾
   - 范例："**这两年是你的音乐成人礼。** 从少年的贪婪变成成年人的选择。"

**绝对禁止**：
- 把纪元写成"XX年你听了A、B、C，XX年你听了D、E、F"的流水账
- 对每个歌手只有一句话的泛泛介绍
- 忽略人生阶段的对应关系

---

### 第3章：音乐琴弦（500-900字）

**开头固定句式**：
```
如果你的灵魂是一把琴，它有X根弦。每一根都在不同的时刻被拨动，发出不同的共振。
```

**每根弦的格式**：
```
**第X根弦：XX弦。** [代表歌手/歌曲列举]。[功能描述——这根弦满足什么情感需求]。[与人生的对应——什么时候这根弦会被拨动]。[一句粗体总结]。
```

**命名原则**：
- 弦的名字必须是"XX弦"的2-3字概括
- 好的命名：史诗弦、孤独弦、真诚弦、温柔弦、故乡弦、生命力弦
- 坏的命名：摇滚弦、流行弦、日语弦（太类型化，没有情感深度）

**深度要求**：
- 不是简单的"你喜欢后摇所以有一根后摇弦"
- 而是"后摇教会你的最重要的事：**有时候不需要说出来，看着就好。**"
- 每根弦必须揭示一个关于用户性格/情感模式的洞察
- 对比不同弦之间的关系（如"后摇是教堂，贰佰是从教堂出来之后喝的第一口酒"）

**弦的数量**：5-6根。不能少于5根，不能多于7根。

---

### 第4章：灵魂疆域（500-800字）

**开头固定句式**：
```
一份歌单画出的不只是品味，还是一张精神地图——你的灵魂住在哪里，去过哪里，想去哪里。
```

**坐标类型**（选取3-5个）：
- **地理坐标**：从歌手地域分布提取（如"北方"、"日本"、"北欧"、"英伦"）
- **时间坐标**：从收藏时间分布提取（如"深夜"——0-4点的收藏）
- **空间坐标**：从歌曲特征提取（如"长歌的疆域"——超过7分钟的歌）
- **情感坐标**：从歌曲主题提取（如"孤独大陆"、"温柔角落"）

**每个坐标的写法**：
```
**坐标名。** [场景化描写——这个"地方"的气质是什么]。[具体歌手/歌曲作为证据]。[为什么用户的灵魂住在这里——与性格/经历的关联]。[粗体金句收束]。
```

**范例**：
```
**深夜。** 你有148首歌是在凌晨0点到4点之间收藏的。深夜是你歌单里的一个独立国度——它有自己的法则、自己的气候、自己的居民。白天的你听后摇和金属，像穿着铠甲在世界里行走。深夜的你听治愈系、钢琴曲——**白天搭建城堡的人，深夜允许自己做一个没有城堡也没关系的人。**
```

---

### 第5章：峥嵘岁月（400-600字）

**格式**：按年龄段快速回顾，每段3-5行。

**每段结构**：
```
**XX-XX岁（YYYY-YYYY）：[一句话主题]。** [2-3句具体支撑，引用关键数据和歌手]。
```

**范例**：
```
**21-22岁（2015-2016）：打开世界的年纪。** 那时候你什么都要——因为你还不知道自己会成为什么样的人。871首歌的贪婪背后是一种焦虑：如果不赶紧把所有的美都抓住，它们会不会消失？
```

**最后一行**（必须是粗体，总结全部弧线）：
```
**你用11年完成了一条完整的弧线：从"我是谁"到"我知道我是谁"再到"我接受我是谁"。**
```

---

### 第6章：下一纪元 —— 音乐预言（800-1500字）

**这是整篇报告中最考验写作功力的章节。**

**结构**：
1. 纪元标题 + 2-3句阶段描述
2. 5-8个独立的"场景推荐块"，用分隔线 `---` 隔开
3. 一段200-300字的哲学释义收尾

**场景推荐块的写法（核心规范）**：

每个块的结构：
```
---

**[场景描写——一句具体的、有画面感的人生瞬间]**

这个时刻需要——[歌手]《[歌名]》。

[为什么是这首歌——3-5句解释，必须连接到用户已有的品味脉络]。[粗体金句结尾]。
```

**范例**：
```
---

**凌晨两点，孩子终于睡了。你一个人坐在客厅，不想看手机，只想听点什么。**

这个时刻需要——坂本龍一《Merry Christmas Mr. Lawrence》。

你听过久石让的温暖，那是"世界是好的"；听过Roberto Cacciapaglia的冥想，那是"世界是空的"。坂本龍一在这两者之间——他的钢琴不治愈你也不超度你，只是坐在你旁边，什么都不说。33岁的父亲独处的深夜，不需要被拯救。**只需要有人证明安静不是虚度。**
```

**推荐逻辑**：
- 60% 自然延伸：基于用户最爱的类型推荐同类优质作品
- 30% 温和探索：在用户已展现的变化趋势上往前推一步
- 10% 灵魂补给：该年龄段普遍需要的情感滋养

**场景设计原则**：
- 场景必须具体到时间/地点/动作（"凌晨两点孩子终于睡了"而不是"安静的夜晚"）
- 场景必须与用户当前年龄+2到+5年的人生阶段匹配
- 如果用户有孩子相关收藏（如儿童歌曲），推测育儿场景
- 如果用户有职场相关收藏（如力量型音乐），推测职业压力场景
- 包含一个"生日"场景和一个"深夜"场景（与报告主题呼应）

**推荐曲目要求**：
- 必须是真实存在的歌曲
- 不能推荐用户歌单中已有的歌
- 每首歌必须说清楚"它和你歌单里的XX是什么关系"——不是突兀出现，而是品味脉络的延伸
- 推荐数量：8-12首歌，分布在5-8个场景中（一个场景可以推荐1-2首）

**哲学释义（最后200-300字）**：
- 以"音乐和时间的关系"或"成长的本质"为主题
- 呼应第一章的核心比喻
- 以一句温暖但不煽情的话收束（如"河流不需要知道大海的形状。它只需要继续流。"）

---

### 第7章：尾声（400-800字）

**目标**：给读者一个完整的"你是谁"的人格素描，温暖收尾。

**必须包含**：
1. 总歌数的再次提及（"3438首歌读完了"）
2. 从第一首歌到最近一首歌的时间跨度
3. 一段"我看到的你是这样一个人"的人格素描（4-6个维度）
4. 一个核心判断（如"你不需要被治愈。你只需要被看见。"）
5. 最后一句交给音乐本身（如"这3438首歌，每一首都在说同一件事：**我在这里。**"）

**人格素描的维度**：
- 内心世界的"大小"（"你的内心有一间巨大的音乐厅"）
- 孤独的处理方式（"你不是不会表达，而是选择用音乐代替表达"）
- 温柔与倔强的并存（深夜的柔软 vs 白天的坚守）
- 独立与渴望的矛盾（"擅长一个人"紧挨着"一起"）
- 信仰的本质（"不是宗教信仰，而是美学信仰"）
- 成长的证据（从贪婪到笃定）

**最后3行（固定格式）**：
```
这就是你的音乐编年史。它不是一份数据报告，不是一份品味评测。
它是一面镜子。
镜子里的人，值得被温柔以待。
```

**页脚信息**：
```
*分析基于YYYY年M月至YYYY年M月的完整收藏数据*
*共XXXX首歌 | XXX小时 | XX年 | 一个人的宇宙*
```

## 三、写作风格硬性规范

### 语气
- 第二人称（"你"），全程不变
- 像一个**真正懂音乐**的老朋友——不是高高在上的乐评人，不是数据分析师，而是坐在你对面的人
- 温暖但不煽情（不用"感动""震撼""令人动容"）
- 洞察但不说教（不用"你应该""建议你"）
- 允许偶尔的幽默和调侃
- 允许引用粗口歌词以保持真实感

### 句式特征（必须体现）

**1. 短句制造节奏感**：
- "但恰恰就是同一个人。"
- "这不是怀旧——**这是告别。**"
- "知道自己是谁的人，不需要每天都证明。"

**2. 排比制造力量感**：
- "Nightwish的北欧神话是庇护所、X JAPAN的18分钟钢琴独奏是庇护所、惘闻的后摇是庇护所。"
- "他需要音乐不是为了"听点什么"，而是为了——**确认自己存在的方式。**"

**3. 对比制造张力**：
- "后摇是你的教堂，贰佰是你从教堂出来之后喝的第一口酒。"
- "白天搭建城堡的人，深夜允许自己做一个没有城堡也没关系的人。"
- "21岁听'骑在银龙背上'，觉得是浪漫的幻想；31岁再听，觉得是对现实的回应。"

**4. 反问/设问引导思考**：
- "能听完这首歌的人，内心一定有一片荒原需要被填满。"
- "一个30岁的人收藏儿童歌曲，大概率是因为他的世界里出现了一个小孩。"

**5. 破折号制造停顿和转折**：
- "你收藏的不是'歌'，而是一个又一个可以让自己暂时不必面对现实的精神空间。"
- "——击中你的不是技巧，是那种'世界崩了也要把美弹完'的执念。"

**6. 金句必须加粗**：
- 每个纪元至少2-3个粗体金句
- 金句的特征：简短（一句话）、有洞察力、让人想截图发朋友圈
- 好的金句："**你从来没有在'听音乐'，你一直在'回家'。**"
- 坏的金句："**你的音乐品味非常丰富多样。**"（这是废话）

### 数据使用原则

**正确方式**：
```
871首歌，一个刚发现自己可以拥有整个世界的人，在疯狂地往口袋里装东西。
```

**错误方式**：
```
根据数据分析，2015-2016年共收藏871首歌曲，收藏频率较高。
```

**核心原则**：数据是为故事服务的，不是为了展示你做了分析。数字应该自然地嵌入叙述中，而不是另起一行"报告"。

## 四、故事化技巧（必须使用）

### 1. 高频收藏日故事化
- 找出 `peak_days` 中单日>10首的日期
- 分析当天收藏的歌曲类型/歌手
- 推测场景："2017年2月25日，你一天收了32首动漫歌"→ 推测这是"少年的告别仪式"
- 用"这不是XX——这是YY"的转折句式

### 2. 跨年夜特写
- 检查每年12月31日或1月1日凌晨的收藏
- 如果有，这是最好的故事素材——一个人跨年夜的歌曲选择暴露最真实的内心
- 范例："2019年元旦夜，你收了一组叫'别网恋'的人的歌——一个人的跨年夜。"

### 3. 深夜收藏的"另一人格"
- `late_night_count` 数据：凌晨0-4点的收藏
- 深夜选择的音乐往往与白天截然不同
- 这是刻画"白天的铠甲vs深夜的柔软"的关键素材

### 4. 重复收藏
- 如果同一首歌被收藏两次（尤其是间隔多年），这是**整篇报告最动人的素材**
- "同一首歌，21岁收一次、31岁收一次。你不是忘了你收过——你是带着10年的记忆重新认领了这首歌。"
- 这种细节必须被发现并放在重要位置

### 5. 歌手的"突然爆发"
- 某个歌手在某年突然以大量歌曲集中出现
- 分析原因：是新发现？是被某个事件触发？是情感需求的变化？
- 范例："中岛美雪2015年1首，然后10年空白，2025年突然爆发10首"→ "10年后你终于长成了配得上这些歌的人"

### 6. 歌名的文学性
- 有些歌的歌名本身就是故事（如"虽然只有一瞬间我竟然想和你厮守一生"）
- 不要只分析音乐本身，歌名选择也是品味的一部分
- 范例："歌名本身就是你想说却没说出口的话。"

### 7. 音乐类型的"断裂"
- 如果某年的主要类型和前一年完全不同，这是人生转折的信号
- 从后摇突然转到民谣 = 从内向探索转向需要人声陪伴
- 从器乐突然转到有歌词的歌 = 从独处模式转向渴望连接

## 五、歌手/歌曲解读深度标准

### 什么是"浅"的解读（禁止）：
```
你喜欢Nightwish，说明你对交响金属有好感。
```

### 什么是"深"的解读（要求）：
```
Nightwish给你的是一种许可——世界可以是史诗般的、魔幻的、巨大的。在一个要求你务实、找工作、读书考试的年纪，这种音乐允许你保留一个戏剧化的自我。那个自我不一定每天出现，但它不能死。它一旦死了，你的世界就会变平。
```

### 解读的三层结构：
1. **表层**：你听了什么（歌手/歌曲名，必须具体）
2. **功能层**：这首歌在你的生活中扮演什么角色（庇护所？盔甲？镜子？药？）
3. **灵魂层**：这个选择揭示了你性格/内心世界的什么秘密

**每个被重点提及的歌手，必须完成这三层解读。**

## 六、人生阶段对应表

| 年龄段 | 典型特征 | 音乐功能 | 关键词 |
|--------|----------|----------|--------|
| 18-22 | 大学/初入社会 | 探索、贪婪、身份寻找 | 什么都想听、量大、类型杂 |
| 23-25 | 确认身份 | 从试探到确认，开始有"主旋律" | 密集期、身份确认、告别少年 |
| 25-28 | 职场磨砺 | 力量型音乐增加，速度下降 | 稳定剂、盔甲、药箱 |

| 31-35 | 和解 | 接受自我，不再证明 | 和解、重听旧歌、允许变化 |
| 35+ | 允许 | 不再"成为"什么，允许"正在成为" | 留白、无所事事的权利 |

## 七、排版规范

### Markdown格式
- 一级标题（#）：仅用于报告总标题
- 二级标题（##）：章节标题（"你的音乐内核""第一纪元"等）
- 三级标题（###）：不使用（保持层次简洁）
- 分隔线（---）：章节之间、推荐场景之间
- 粗体（**）：金句、数据开头句、弦的名字
- 不使用斜体（影响中文阅读）
- 不使用无序/有序列表（保持散文感）

### 段落长度
- 每段3-8行
- 不允许单句成段（除非是有意为之的节奏控制，如"但恰恰就是同一个人。"）
- 长段落（>10行）必须拆分

### 引用歌名格式
- 中文歌：《歌名》
- 日文/外文歌：《歌名》或不加书名号直接引用
- 歌手名：不加任何标点，直接使用

## 八、质量自检清单

写完报告后，用以下清单检查：

- [ ] 总字数是否达到要求？
- [ ] 每个纪元是否有具体的日期故事（不能全是泛泛而谈）？
- [ ] 粗体金句是否每500字至少出现1个？
- [ ] 是否有3个以上的"具体歌名+具体时长/特征"被引用？
- [ ] 是否有对比句式（白天vs深夜、21岁vs31岁、后摇vs民谣）？
- [ ] 推荐章节的每首歌是否有场景描写？
- [ ] 是否有至少一个"重复收藏"或"跨年夜"的故事？
- [ ] 尾声是否有人格素描（而不只是"祝你听歌愉快"）？
- [ ] 是否全程避免了"根据数据""分析显示"等生硬表述？
- [ ] 是否每个歌手都做了三层解读（表层→功能层→灵魂层）？

## 九、常见错误（绝对禁止）

1. ❌ "你的音乐品味非常丰富" → ✅ 用具体比喻代替
2. ❌ "根据数据分析，你在2017年收藏了562首歌" → ✅ "562首歌，23岁的灵魂最饥饿的时期"
3. ❌ "你可能会喜欢坂本龍一" → ✅ 构建具体场景后自然引入
4. ❌ 对所有歌手的介绍都是"XX是一位著名的YY歌手" → ✅ 不介绍歌手，直接说他对你意味着什么
5. ❌ 每个纪元结构完全一样（数据→歌手→总结） → ✅ 变化叙述节奏
6. ❌ 推荐章节变成"推荐列表"（歌名+一句推荐语） → ✅ 必须是场景化叙事
7. ❌ 尾声只有3行感谢语 → ✅ 至少400字的人格素描
8. ❌ 忽略深夜收藏、高频日等有故事感的数据 → ✅ 这些是报告的灵魂
9. ❌ 全篇没有一个短句制造节奏（全是长句铺排） → ✅ 长短交替
10. ❌ 琴弦章节只有3根弦 → ✅ 必须5-6根

## 十、HTML 报告设计规范

当需要手工编写精美 HTML 报告时（超越 Markdown→HTML 自动转换），遵循以下设计语言：

### 页面结构
- 每个章节独占一页（`page-break-after: always`）
- 每页有独立的背景色/渐变
- 每页有 2-3 个轻量装饰元素（CSS伪元素实现）
- 每页底部有页码

### 颜色主题（按章节）
- 封面：暖橙渐变 + 唱片/音符装饰
- 音乐内核：浅紫渐变 + 声波装饰
- 各纪元：分别用暖金、红橙、嫩绿、琥珀、淡蓝
- 琴弦：浅紫 + 竖线装饰
- 灵魂疆域：米黄 + 网格/指南针
- 峥嵘岁月：暖棕 + 时间线/沙漏
- 尾声：暖金渐变 + 光芒/心形

### 排版组件
- `.quote-block`：引用块（左侧色条 + 浅色背景）
- `.string-item`：琴弦卡片（左侧彩色边框）
- `.territory-card`：疆域卡片（圆角 + 微边框）
- `.timeline-item`：时间轴项（左侧圆点 + 浅色背景）

### 字体
- 标题：Noto Serif SC (衬线)
- 正文：Noto Sans SC (无衬线)
- 通过 Google Fonts CDN 引入
