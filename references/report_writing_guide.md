# 音乐编年史报告写作指南

## 报告整体结构

报告应该包含以下部分，按顺序编写：

### 1. 音乐内核总述（开篇）
- 用一个生动的比喻概括用户的音乐灵魂
- 点明贯穿始终的审美线索
- 从第一首歌的收藏日切入，揭示灵魂底色
- 字数：400-600字

### 2. 编年史纪元（主体，按年份分段）
- 将用户的收藏历史划分为3-6个"纪元"
- 每个纪元以年份+核心特征命名（如"第一纪元：2015-2016 ——饥饿的灵魂"）
- 每个纪元包含：
  - 数据概览（首数、日均收藏量）
  - 核心发现（最爱歌手、新出现的音乐类型）
  - **关键时刻故事化**：挑出有故事感的具体日期（如跨年夜、某个高频收藏日），描写当时的场景和心理
  - 个性化解读（不是数据罗列，而是洞察）
- 每个纪元400-800字

### 3. 深层解读（升华部分）
- 提炼3-5条贯穿全歌单的"线索"
- 线索举例：
  - 长歌信仰（超长歌曲的收藏偏好）
  - 重复收藏（同一首歌多次收藏的意义）
  - 深夜收藏（凌晨0-4点的收藏习惯）
  - 语言偏好变迁
  - 某种特定的情绪模式
- 每条线索150-300字

### 4. 灵魂画像（总结）
- "如果你的灵魂是一把琴"或类似比喻
- 将用户的音乐品味抽象为3-6根"弦"或"色彩"
- 每根弦对应一类音乐和背后的情感需求
- 字数：400-600字

### 5. 尾声
- 从第一首歌到最近一首歌的呼应
- 温暖且有力量的收尾
- 字数：200-400字

## 写作风格要求

### 语气
- 第二人称叙述（"你"）
- 像一个懂音乐的老朋友在和用户深谈
- 温暖但不煽情，洞察但不说教
- 允许偶尔的幽默和调侃

### 数据使用
- 数据必须精确（直接引用分析结果中的数字）
- 但数据是为故事服务的，不是为了展示
- 避免"根据数据分析..."这种生硬的表达
- 用具体的场景和细节替代干巴巴的数据罗列

### 故事化技巧
- 挑出"高频收藏日"（单日收藏>10首），猜测当时的场景
- 关注跨年夜、特殊日期的收藏
- 关注凌晨收藏——深夜的选择往往更诚实
- 关注重复收藏——同一首歌收两次意味着什么
- 关注"突然出现"的歌手——大量集中收藏某个歌手是什么信号

### 结合用户年龄
- 根据用户提供的年龄，推算每个年份对应的人生阶段
- 将音乐选择与人生阶段结合解读
  - 大学时期：探索、贪婪、什么都想尝试
  - 初入职场：需要力量型音乐
  - 中期：从探索转向确认，速度放缓
  - 成熟期：回望、和解、接受

### 个性化原则
- **绝对不要使用模板化的表述**
- 每一段解读必须基于该用户独特的数据
- 如果用户补充了个人信息，要有机地融入分析
- 关注歌单中的"异常值"——它们往往是最有故事的部分

## 报告长度

- 完整报告：3000-6000字
- 如果用户歌单较小（<500首），适当缩短至2000-3000字
- 如果歌单巨大（>3000首），可以适当扩展，但不超过8000字

## 关键数据解读维度

基于 analysis_data.json，重点关注：

1. **year_counts** → 年度收藏量的起伏（哪年最多？哪年断崖？为什么？）
2. **top_artists** → 最爱歌手（前20名必须逐一分析含义）
3. **year_top_artists** → 各年份偏好变化（品味的演进路径）
4. **lang_distribution** → 语言分布（什么比例的日/英/中/韩？说明什么？）
5. **duration_stats** → 时长偏好（喜欢长歌的人和喜欢短歌的人完全不同）
6. **peak_days** → 高频收藏日（那天发生了什么？）
7. **late_night_count** → 深夜收藏量
8. **earliest_songs / latest_songs** → 首尾对比（成长轨迹）
9. **decade_distribution** → 听的音乐来自哪个年代（是否怀旧型？）
